# Java Reflection

Reflection in Java allows you to inspect, analyze, and manipulate classes, methods, and fields at runtime, even if you don't know their exact structure at compile time. This mechanism is especially useful in cases where you need to build more dynamic applications.

## Commands used in this section:

To generate java class:
```
javac ReflectionInspector.java
```
To execute java class:
```
java ReflectionInspector
```
To execute java class with arg:
```
java ReflectionInspector java.util.Calendar
```

***NOTE: This concept is used a lot in frameworks and IDE`s, for example when you are in IntelJIdea or Eclipse it has the feature of auto-completing methods, fields, classes etc, for this it uses reflection.***
