# Course: Java from the Ground Up  
**Master the Ecosystem Without IDEs**  

## Description  
This course is designed for programmers with prior experience who aim to understand how Java works internally, performing tasks manually that are typically automated by tools like IDEs, frameworks, or build systems. From compiling code to creating libraries and working with bytecode, you will learn to master the Java ecosystem in a deep and technical way.  

---

## Syllabus  

### 0. Java Fundamentals  
- What is the JDK, JRE, and JVM  
- Relationship between JDK, JRE, and JVM  
- Key Tools in Java  
- Advantages of Java Environment Design  
- Overview of the Life Cycle of a Java Program  

### 1. Manual Compilation and Execution  
- Create, compile, and run your first program:  
  - Basic usage of `javac` and `java`.  
  - Analysis of the generated `.class` file.  
- Disassemble classes with `javap`.  
- Inspecting the generated bytecode.  
- Advanced options for `javac` and `java`.  

### 2. Project and Package Structuring  
- Introduction to Packages in Java (`package` and `import`).  
- Naming Conventions for Packages.  
- Manually creating a project structure:  `src/com/company`
- Compilation while maintaining package structure (`javac -d`).  

### 3. Creation and Use of Libraries  
- Generating a JAR File:  
- Using `jar cf` to create libraries.  
- Configuring `CLASSPATH` to use external libraries.  
- Creating an Executable JAR with a Custom Manifest (`Manifest.mf`).  

### 4. Reflection and Dynamic Class Loading  
- Using the Reflection API (`java.lang.reflect`).  
- Inspecting Classes and Methods at Runtime.  
- Practical Example: Loading Plugins Dynamically at Runtime.  

### 5. REST API Without Libraries
- Building a REST API using only JDK classes (`HttpServer`).
- Defining routes and handlers manually (`/health`, `/users`).
- Processing request methods, headers, and JSON body manually.
- Returning JSON responses with proper HTTP status codes.

### 6. Custom Annotations  
- Creating custom annotations with `@interface`.  
- Using `@Retention` and `@Target` meta-annotations.  
- Processing annotations at runtime with reflection.  
- Practical Example: Building a simple annotation scanner (like a mini-framework).  

### 7. REST API with Annotations (Mini Framework)
- Defining endpoints with `@Route(path, method)`.
- Scanning controllers with reflection to register routes dynamically.
- Dispatching HTTP requests to controller methods using an annotation-based route registry.
- Comparing manual routing (module 12) vs declarative routing with annotations.
